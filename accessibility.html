// 01 — accessibility statement

Accessibility statement.

Last updated: June 2026.

// 02 — our commitment

Our commitment

We claim accessibility as a core value. This site, and everything we build, targets WCAG 2.2 AA as a minimum. The site has been built and tested with this in mind.

// 03 — what we've done

What we’ve done

  • Semantic HTML throughout.
  • Full keyboard operability.
  • Visible focus indicators on every interactive element.
  • AA-or-better contrast on every text and background combination we use.
  • Alt text on meaningful images and empty alt on decorative ones.
  • Labelled forms with accessible error messages.
  • A skip-to-content link as the first focusable element.
  • Respects your reduced-motion preference.
  • Readable at 200% zoom with no horizontal scrolling.
  • Tested with NVDA and VoiceOver manually before launch.

// 04 — known issues

Known issues

We’re being honest. Where we know about a problem we haven’t fixed yet, we list it here. At time of writing: no known issues at time of publication, June 2026. This section will be updated as we find or fix things.

// 05 — how to report a problem

How to report a problem

If you find something that doesn’t work for you, please tell us. Email hello@birminghamtech.collective with ‘Accessibility’ in the subject line and we’ll get back within five working days.

The standards we reference are WCAG 2.2, the Equality Act 2010, and BS 8878.

A not-for-profit forming as a community interest company in Birmingham. We build accessible software with and for our community.
